# C Call Hierarchy Explorer

`.c`와 `.h` 파일을 Tree-sitter로 빠르게 파싱하고, libclang으로 실제 심볼 참조를 보강한 뒤 PySide6 가상 스크롤 UI에 호출 단계를 표시합니다.

현재 개발 소스와 배포 버전은 `1.1.17`입니다. `Python\배포\C Call Hierarchy Explorer 1.1.17` 폴더의 배포 파일에는 PySide6, Tree-sitter, tree-sitter-c, libclang이 포함되므로 대상 PC에 Python을 별도로 설치할 필요가 없습니다.

## 설치 배포본

- `C-Call-Hierarchy-Explorer-Setup-1.1.17.exe`: 현재 사용자 계정에 설치하고 바탕 화면·시작 메뉴 바로가기를 생성합니다.
- `C-Call-Hierarchy-Explorer-Portable-1.1.17.exe`: 설치 없이 실행하는 포터블 버전입니다.
- Windows 설정의 **설치된 앱**에서 제거할 수 있습니다.
- `packaging\build_release.ps1`은 실행 파일 진단, 설치 파일 생성, SHA-256 파일 생성을 자동으로 수행합니다.

## 개발환경 설치

1. Windows용 Python 3.10 이상을 설치합니다. 설치 화면에서 **Add Python to PATH**를 선택하십시오.
2. 이 폴더의 `install.bat`를 한 번 실행합니다.
3. 설치가 끝나면 `run.bat`를 실행합니다.

명령 프롬프트에서 직접 설치하려면 다음과 같습니다.

```bat
cd /d "C:\Users\USER\Documents\HTA 프로그램\Python\프로그램"
py -3 -m venv .venv
.venv\Scripts\python.exe -m pip install --upgrade pip
.venv\Scripts\python.exe -m pip install -r requirements.txt
```

설치되는 핵심 패키지는 다음과 같습니다.

- `PySide6`: Qt 기반 반응형 데스크톱 UI
- `tree-sitter`, `tree-sitter-c`: C 구문 트리와 함수/호출식 추출
- `libclang`: 컴파일 데이터가 있을 때 정확한 함수 정의와 호출 대상 연결

`libclang` PyPI 패키지는 필요한 Clang 공유 라이브러리를 포함하므로 일반적인 Windows 환경에서는 LLVM을 별도로 설치할 필요가 없습니다. 프로젝트에 `compile_commands.json`이 있으면 자동으로 찾아 해당 컴파일 옵션을 사용하고, 없으면 GNU C11 기본 옵션으로 보강 분석합니다.

## 사용법

1. `run.bat`를 실행하고 시작 화면이나 **파일 → 폴더 열기**를 선택합니다.
2. `.c`와 `.h` 파일이 모두 분석되며 `main()` 하나를 기본 MAIN LOOP 루트로 선택합니다.
3. ISR/IRQ/Handler 계열 진입점은 별도 영역에서 각각 1단계부터 표시됩니다.
4. 상단의 **시작점** 목록에서 중복된 `main()` 후보를 직접 변경할 수 있습니다.
5. **보기** 메뉴에서 파일/함수 트리, CODE 미리보기, 기타 독립 시작점을 켤 수 있습니다. 파일/함수 트리와 CODE 미리보기의 체크 상태는 다음 실행에도 유지됩니다.
   파일/함수 패널의 폭을 넓히면 `선택폴더\하위경로\main.c` 전체를 표시하고, 좁히면 모든 파일에 동일한 수의 상위 경로 단계를 적용한 뒤 파일명을 우선 보존해 `..\main.c`, `..\m..n.c` 형태로 자동 축약합니다. 파일/함수 보기가 활성화된 동안 분할 손잡이는 `..\` 뒤 10글자를 확인할 수 있는 최소 폭 아래로 줄어들거나 완전히 접히지 않습니다.
6. **Excel 트리 출력**은 현재 화면의 호출 트리와 전체 함수 목록을 `.xlsx`로 저장합니다. 기본 파일명은 현재 소스 폴더명으로 시작합니다.
   같은 이름의 파일이 이미 있으면 기존 파일을 덮어쓰지 않고 `_1`, `_2` 순서로 사용 가능한 이름을 자동 선택합니다.
   저장이 완료되면 방금 생성한 Excel 파일을 Windows 기본 연결 프로그램으로 자동 실행합니다.
7. 호출 트리의 작은 `+ / −` 상자를 누르거나 함수 행을 더블클릭하면 하위 호출을 접고 펼칠 수 있습니다.
   자동 갱신 후에도 사용자가 접어 둔 가지는 접힌 상태를 유지하고, 펼친 가지의 변경만 즉시 화면에 나타납니다.
   단계 머리글에는 클릭 시 실행될 `모두 접기` 또는 `모두 펼치기`가 표시되며, 머리글을 누르면 해당 단계의 모든 하위 트리를 한 번에 접거나 펼칩니다.
   부모 함수 오른쪽 첫 빈 셀에는 해당 부모의 직접 자식 전용 `모두 접기` 또는 `모두 펼치기` 버튼이 표시됩니다. 이 버튼은 다른 부모 가지나 더 아래 단계의 기존 접기 상태를 변경하지 않습니다.
8. 성공적으로 분석한 폴더는 최근 이력에 저장됩니다. 시작 화면에는 최근 5개만 간략히 표시되고 전체 목록은 **파일 → 최근 폴더**에서 확인하거나 지울 수 있습니다.
9. 함수 셀을 선택한 뒤 `↑/↓`로 행 이동, `→`로 펼치기 또는 첫 자식 이동, `←`로 접기 또는 부모 이동, `Enter/Space`로 접기·펼치기를 할 수 있습니다.
   선택 셀은 내용과 연결선을 가리지 않는 반투명 채움과 파란 테두리로 표시되고, 부모 셀은 화면과 Excel에서 자식 범위 전체를 세로 병합한 형태로 표시됩니다.
   셀 폭보다 긴 함수명·파일명·행 번호는 말줄임표로 표시되며 해당 셀 위에 마우스를 올리면 전체 문자열이 툴팁으로 나타납니다.
   외부/미확인 함수를 선택하면 CODE 미리보기에 현재 분석 폴더에서 정의를 찾지 못했다는 안내가 표시됩니다.
   CODE 미리보기 상단에는 부모 함수, 정의가 확인된 자식 호출, 외부/미확인 호출 목록이 표시되며 코드 안의 함수명도 각각 파란색, 초록색, 주황색으로 강조됩니다.
10. 검색 상자에 함수명 또는 파일명을 입력하면 현재 트리를 재구성하지 않고 첫 번째 일치 셀로 이동합니다. `◀/▶` 버튼이나 Enter로 이전·다음 일치 항목을 순회합니다. 검색 대상이 접힌 가지 안에 있으면 대상이 보이는 부모 경로까지만 펼치고, 대상 자체와 그 아래 가지의 기존 접기 상태는 유지합니다.
11. 프로젝트를 처음 분석한 뒤 분석 데이터와 트리 접기 상태, 선택 셀, 스크롤 위치, 검색어, 패널 표시 및 크기, 파일 트리 전개 상태를 프로젝트별 캐시 파일에 기록합니다. 다시 열 때는 마지막 트리를 먼저 표시하고 곧바로 파일 수정시각·크기를 검사해 변경된 파일만 최신화합니다.
    캐시는 상태나 분석 결과가 달라진 경우에만 저장 대상으로 표시되며 5분 주기 또는 프로그램 종료·다른 폴더 전환 시 기록합니다. 2초 자동 감시는 캐시 파일을 저장하지 않습니다.
12. 메인 메뉴의 **설정 → 프로젝트 설정 열기…** (`Ctrl+,`)에서 VS Code 형태의 체크박스 폴더 트리로 분석 범위를 지정합니다. 체크한 폴더의 `.c/.h` 파일만 함수 목록·호출 트리·파일/함수 보기·CODE 미리보기·변경 감시에 포함되며, 체크를 해제한 가지와 모든 하위 폴더는 제외됩니다. 별도의 **외부/미확인 함수 표시** 옵션으로 정의를 찾지 못한 호출의 트리 및 CODE 구분 표시 여부를 선택할 수 있습니다. 설정 화면의 체크박스와 글자는 어두운 배경에서 식별할 수 있는 고대비 색을 사용하고, 선택 행은 내용이 가려지지 않는 반투명 파란색으로 표시합니다. 폴더 트리는 펼칠 때만 읽어 대형 프로젝트에서도 설정 창의 응답성을 유지하고, 선택 상태와 표시 옵션은 프로젝트별로 저장됩니다. 설정 검색은 공백으로 구분한 각 검색어가 설정 키워드에 포함되는지를 즉시 확인하며 한 글자 검색도 지원합니다. 일치 항목이 없으면 빈 화면 대신 안내와 `검색어 지우기` 버튼을 표시하고, Enter를 눌러도 체크 상태나 적용 상태를 변경하지 않습니다. 설정 변경도 캐시 트리를 먼저 유지한 채 달라진 폴더의 파일만 증분 반영합니다.
13. **도움말 → 업데이트 확인…**은 하루 한 번 공개 GitHub Releases에서 정식 최신 버전을 백그라운드로 확인합니다. 새 버전은 사용자가 승인한 경우에만 설치 파일을 다운로드하며, GitHub가 제공한 파일 크기와 SHA-256이 모두 일치해야 설치를 실행합니다. **업데이트 다운로드 검증…**은 현재 최신 버전과 같아도 전체 설치 파일을 내려받아 검증한 뒤 테스트 파일을 삭제하므로 배포 서버 연결을 확인할 수 있습니다.

## 동작 및 성능

- 자동 감시는 2초마다 전체 파일 내용을 읽지 않고 파일 경로와 수정 시각만 비교하며, 상태 표시줄에 분석 범위 안에서 확인된 `마지막 변경 Date`를 함께 표시합니다.
- 같은 함수가 여러 번 호출되면 호출 위치별로 각각 트리에 표시하며, 수정 시각과 파일 크기 메타데이터 변경을 확인합니다.
- 중복 함수도 각각 하위 호출을 전개하고 최대 단계는 실제 호출 관계에서 계산합니다. 단계 수 자체에는 제한을 두지 않으며 순환 호출만 해당 경로에서 종료합니다. 대규모 프로젝트의 기하급수적 조합으로 인한 메모리 고갈을 막기 위한 전체 표시 행 안전장치는 유지됩니다.
- 한 부모 함수 안에서 같은 함수를 여러 번 호출하면 트리 노드는 하나로 합치고, 호출 행이 서로 다르면 `파일명:23, 24, 25` 형식으로 표시합니다. 서로 다른 부모나 호출 경로의 같은 함수는 별도 노드로 유지합니다.
- 변경 또는 추가된 파일만 Tree-sitter로 다시 파싱하고 삭제 파일만 캐시에서 제거합니다.
- 프로젝트 캐시는 사용자 LocalAppData의 앱 전용 `project-cache` 폴더에 원자적으로 저장되며 소스 프로젝트 폴더에는 별도 파일을 만들지 않습니다.
- 분석과 Excel 생성은 GUI 스레드가 아닌 작업 스레드에서 실행됩니다.
- 호출 트리는 전체 행에 위젯을 만들지 않고 현재 보이는 행만 그려 대규모 트리의 스크롤 부하를 줄입니다.
- 접힌 하위 트리의 마지막 행을 미리 계산해 두므로 단계 전체 접기와 화면 갱신 시 숨겨진 자손을 한 행씩 다시 순회하지 않습니다.
- 마우스 휠은 1틱마다 함수 행 1칸씩 이동합니다.
- Excel 출력 중에는 넓은 진행 표시줄에 호출 행 준비, 함수 목록 생성, 압축 저장 진행률이 표시됩니다. 호출 관계 시트는 모든 호출을 펼친 상태로 기록하며 부모 셀은 자식 범위까지 세로 병합되고 말단 뒤쪽 빈 단계는 회색으로 수평 병합됩니다.
- 고정 단계 머리글은 현재 화면에 가장 가까운 호출 경로를 `1단계 main()`, `2단계 App_Task()` 형식으로 유지합니다.
- 64KB 이상의 전처리기/상수 테이블 밀집 파일은 정지 방지를 위해 선형 C 함수 스캐너로 추출하고, 일반 파일은 Tree-sitter로 분석하며 libclang이 호출 대상을 보강합니다.
- 독립 루트는 선택된 `main()`에서 도달 가능한 함수를 제외합니다. 인터럽트 루트는 표준 예외 핸들러, `IRQHandler`, `ISR/IRQ` 명명 또는 interrupt 속성이 확인된 프로젝트 함수만 채택하고, Drivers/HAL/CMSIS/SDK/라이브러리·예제·테스트 경로는 제외합니다. 기타 독립 루트는 호출자가 없고 `Task/Thread/Callback/Hook/Worker` 등 명확한 진입점 이름을 가진 비정적 함수만 표시합니다.

## 참고

- 매크로가 함수 호출을 생성하거나 함수 포인터가 런타임에 바뀌는 경우 정적 분석만으로 대상이 확정되지 않을 수 있습니다.
- 가장 정확한 결과를 위해 빌드 시스템에서 `compile_commands.json`을 생성해 소스 루트 아래에 두는 것을 권장합니다.
- Excel 출력은 별도 Excel Python 패키지 없이 표준 `.xlsx` 형식으로 생성됩니다.
