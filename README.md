# C 코드 함수/호출 트리 HTA

Windows에서 폴더 안의 `.c`, `.h` 파일을 분석하여 함수 목록과 호출 관계를 클릭형 트리로 보여주는 HTA 프로그램입니다. Microsoft Excel이 설치되어 있으면 동일한 결과를 윤곽선 그룹이 포함된 `.xlsx` 파일로 출력합니다.

## 실행

1. `CCodeTree.hta`를 더블 클릭합니다.
2. **폴더 선택**을 눌러 C 소스가 있는 폴더를 지정합니다.
3. 기본 화면에서 `main`과 인터럽트 시작점 기준으로 완전히 전개된 호출 구조를 확인합니다.
4. **보기** 메뉴에서 파일/함수 트리 또는 소스 미리보기를 필요할 때만 켭니다.
5. **Excel 트리 출력**을 누르면 선택한 폴더에 `C_코드_트리_날짜_시간.xlsx`가 생성됩니다.

업데이트 후에는 기존 HTA 창을 닫고 다시 실행하십시오. 현재 버전은 제목 표시줄과 상단 오른쪽에 `v2.3`으로 표시됩니다.

## 자동 반영 방식

- 폴더를 처음 선택할 때 `.c`, `.h` 파일 전체를 한 번 읽고 분석합니다.
- 이후에는 2초마다 파일 경로와 `DateLastModified` 메타데이터만 검사합니다. 파일 크기나 내용 비교는 하지 않습니다.
- 수정 날짜가 바뀐 파일만 다시 읽고 다시 파싱합니다.
- 새 파일은 추가 분석하고 삭제된 파일은 메모리 캐시에서 제거합니다.
- 바뀌지 않은 파일의 내용은 다시 읽거나 다시 파싱하지 않습니다.
- 편집기에서 파일을 **저장한 시점**에 UI가 자동 갱신됩니다.
- `.git`, `.svn`, `.hg`, `node_modules`, `dist`, `build`, `.vs` 폴더는 제외합니다.
- 백그라운드 자동 감시는 진행 표시줄을 사용하지 않고 상태 문구로만 결과를 표시합니다.

## 호출 구조 표시 기준

- `main`, `app_main`, `main_loop`, `main_task` 등을 메인 실행 시작점으로 분류합니다.
- `main()` 후보가 여러 개면 예제·테스트·외부 라이브러리 경로를 낮게 평가하여 기본 메인 루트 하나만 선택합니다. 상단 **메인 루트** 목록에서 다른 `main.c:줄`을 직접 선택할 수 있습니다.
- 선택하지 않은 다른 `main()` 후보는 기타 독립 시작점에 중복 표시하지 않습니다.
- 이름과 선언에 `ISR`, `IRQ`, `IRQHandler`, `_Handler`, `interrupt`가 포함된 함수는 인터럽트 독립 시작점으로 분류합니다.
- 호출자가 없는 함수와 메인·인터럽트에서 도달하지 못한 고립 구성요소는 기타 독립 시작점으로 표시합니다.
- 시작 함수가 1단계이며 하위 호출은 2단계, 3단계 순으로 표의 열에 표시됩니다. 단계별 열에는 구분선과 교차 배경색이 적용됩니다.
- 메인, 각 인터럽트, 기타 독립 루트는 서로 다른 테두리 영역으로 분리되며 각 영역은 다시 1단계부터 시작합니다.
- 호출 구조는 처음부터 전부 펼쳐서 표시합니다. 이미 전개한 함수는 `위에서 전개됨`, 재귀 호출은 `순환 호출`로 표시해 무한 반복을 방지합니다.
- 같은 이름의 함수 정의가 백업·예제·다른 프로젝트에 여러 개 있으면 호출자와 공통 폴더 경로가 가장 긴 정의를 선택합니다. 따라서 같은 프로젝트 내부의 `App_Task()`와 하위 호출을 계속 추적할 수 있습니다.
- 단계 표시는 각 함수 옆에 반복하지 않고 호출 트리 상단의 고정 행에만 표시합니다.
- 상하 스크롤 시 현재 위치에서 가장 가까운 호출 경로가 `1단계 main() | 2단계 App_Task() | ...` 형식으로 상단에 고정됩니다. 좌우 스크롤 시에도 단계 열과 함께 이동합니다.
- 각 행마다 깊이 수만큼 빈 표 셀을 만들지 않는 경량 좌표형 렌더링을 사용합니다. 새 트리는 화면 밖에서 완성한 뒤 한 번에 교체하므로 자동 갱신 중 기존 화면이 하얗게 지워지지 않습니다.
- 여러 독립 루트가 같은 하위 함수를 공유하면 첫 전개 이후에는 참조로 표시하여 중복 DOM과 메모리 사용을 줄입니다.
- 기본 화면은 선택한 메인 프로젝트의 메인 트리와 ISR/IRQ 진입점만 생성합니다. 호출자가 없는 모든 함수가 필요할 때만 **보기 → 기타 미호출 루트 표시**를 켭니다. 대형 프로젝트에서는 이 옵션을 끈 상태가 권장됩니다.

## 진행 표시와 UI 응답

- 폴더 검색, 파일 파싱, 호출 관계 연결, 파일 트리 구성, 호출 트리 구성을 각각 별도 단계로 표시합니다.
- 현재 처리한 파일·함수 수와 작업 중인 경로가 진행 표시줄에 나타납니다.
- 한 번에 긴 작업을 실행하지 않고 약 20~25ms 또는 정해진 항목 묶음 단위로 처리합니다.
- 각 묶음 사이에 HTA 메시지 처리를 재개하므로 대규모 폴더를 분석할 때도 창 이동과 화면 갱신이 계속 처리됩니다.
- 2초 자동 감시가 꺼져 있어도 **지금 변경 확인** 버튼으로 한 번만 검사할 수 있습니다.

## Excel 출력

- `함수 트리`: 파일 아래에 각 함수가 배치됩니다.
- `호출 관계`: 각 함수를 시작점으로 하위 호출을 표시합니다.
- 행 왼쪽의 `+/-` 윤곽선 버튼으로 트리를 펼치거나 접을 수 있습니다.
- 함수명, 파일, 시작/종료 줄, 선언, 호출 수와 호출자 수를 함께 기록합니다.
- Excel 윤곽선 제한에 맞춰 호출 트리 깊이는 8단계까지 출력합니다.

## 분석 범위와 한계

정적 분석 방식이므로 일반 함수 정의와 직접 호출, 재귀/순환 호출, 다른 `.c` 파일의 고유한 함수 정의를 찾을 수 있습니다. 함수 포인터, 전처리기 매크로로 만들어지는 함수, 조건부 컴파일 결과, 동일 이름의 여러 전역 함수, 런타임에 결정되는 호출은 완전히 판정할 수 없으며 `외부/미확인`으로 표시될 수 있습니다. 정확한 컴파일 단위 분석기가 아니라 코드 탐색용 도구입니다.

## 분석기 테스트

명령 프롬프트 또는 PowerShell에서 다음을 실행합니다.

```powershell
cscript //nologo tests\test_analyzer.js
```
